🔍 Análisis del QR del Certificado

Simulación de lo que debería pasar cuando escaneas el código QR

📱 URL del QR Escaneado

https://tienda.certificalatam.com/web/ultimocertificado/equiposdemedicion/27/42597

🔗 Desglose de la URL

  • Dominio: tienda.certificalatam.com
  • Ruta: /web/ultimocertificado/
  • Tipo de documento: equiposdemedicion
  • ID del equipo: 27
  • ID del usuario/cliente: 42597

✅ Comportamiento Esperado

Cuando escaneas el QR, debería:

  1. Abrir automáticamente el navegador
  2. Acceder a la URL del certificado
  3. El servidor busca el certificado más reciente para:
    • Tipo: "equiposdemedicion"
    • Equipo ID: 27
    • Cliente ID: 42597
  4. Descargar automáticamente el PDF del certificado
  5. Mostrar el certificado en el navegador o en el visor de PDF

❌ Problema Actual

Error: "No se pudo cargar el documento PDF"

Causa: El servidor remoto tienda.certificalatam.com aún tiene el código anterior con el bug del ValueError.

Código problemático:

registro_id = int(strmaquinara_id) # ❌ Falla si strmaquinara_id no es numérico cliente_id = int(userid) # ❌ Falla si userid no es numérico

✅ Solución Implementada

Hemos corregido el código para manejar parámetros inválidos:

# ✅ Conversión segura registro_id = int(strmaquinara_id) if strmaquinara_id and str(strmaquinara_id).isdigit() else False cliente_id = int(userid) if userid and str(userid).isdigit() else False

Mejoras implementadas:

  • ✅ Validación de parámetros obligatorios
  • ✅ Conversión segura de IDs
  • ✅ Logging detallado para diagnóstico
  • ✅ Mensajes de error informativos
  • ✅ Compatibilidad con versiones anteriores de Python
