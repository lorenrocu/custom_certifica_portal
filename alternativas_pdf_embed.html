Alternativas para mostrar PDF embebido (sin iframe)

1. Usando <object> (YA IMPLEMENTADO)

Ventajas: Mejor compatibilidad, fallback automático, más estándar

<object t-att-data="urldownload" type="application/pdf" width="100%" height="600">
    <div>Fallback si no se puede mostrar el PDF</div>
</object>

2. Usando <embed> (Alternativa simple)

Ventajas: Muy simple, compatible con la mayoría de navegadores

<embed t-att-src="urldownload" type="application/pdf" width="100%" height="600" />

3. Visor PDF.js personalizado

Ventajas: Control total, funciona en todos los navegadores, no depende del visor del navegador

<iframe t-att-src="'/web/static/lib/pdfjs/web/viewer.html?file=' + urldownload" width="100%" height="600"></iframe>

4. Botón de descarga con vista previa opcional

Ventajas: Siempre funciona, no depende de capacidades del navegador

<div class="text-center">
    <a t-att-href="urldownload" class="btn btn-primary btn-lg" target="_blank">
        <i class="fa fa-file-pdf-o"></i> Ver Certificado
    </a>
</div>

5. Combinación: Embed + Fallback

Ventajas: Intenta mostrar embebido, si falla muestra botón de descarga

<div id="pdf-container">
    <embed t-att-src="urldownload" type="application/pdf" width="100%" height="600" 
           onerror="document.getElementById('pdf-fallback').style.display='block'; this.style.display='none';" />
    <div id="pdf-fallback" style="display:none; text-align:center; padding:20px;">
        <p>No se puede mostrar el PDF embebido</p>
        <a t-att-href="urldownload" class="btn btn-primary">Abrir PDF</a>
    </div>
</div>

Recomendación

Ya implementé la opción 1 (<object>) que es la más robusta. Si quieres probar otra, puedo cambiarla fácilmente.

Ventajas del <object> vs <iframe>

  • ✅ Mejor manejo de errores
  • ✅ Fallback automático si el PDF no se puede mostrar
  • ✅ Más estándar HTML5
  • ✅ No depende de políticas de iframe (X-Frame-Options)
  • ✅ Mejor accesibilidad
